# SparkYa GitHub Pages Site

Static support and legal pages for the SparkYa app.

## Pages

- `index.html` - landing page with links to all app support/legal pages
- `support.html` - Support URL for App Store Connect or other distribution platforms
- `privacy.html` - Privacy Policy
- `terms.html` - User Agreement / Terms of Service

## Update Before Publishing

Replace the placeholder contact email wherever it appears:

```text
support@sparkya.app
```

If SparkYa collects different data than described in `privacy.html`, update the data categories before publishing. This template is a starting point and is not legal advice.

## Deploy to GitHub Pages

1. Create a GitHub repository, for example `sparkya-support`.
2. Push this project to the repository.
3. In GitHub, open `Settings` -> `Pages`.
4. Set `Source` to `Deploy from a branch`.
5. Select your branch, usually `main`, and folder `/root`.
6. Save. GitHub will publish the site after a short build.

Your URLs will look like:

```text
https://<your-github-username>.github.io/<repo-name>/
https://<your-github-username>.github.io/<repo-name>/support.html
https://<your-github-username>.github.io/<repo-name>/privacy.html
https://<your-github-username>.github.io/<repo-name>/terms.html
```
